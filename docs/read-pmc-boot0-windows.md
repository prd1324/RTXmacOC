# Прочитать PMC_BOOT_0 из Windows (без Мака, без Linux)

Цель: подтвердить декодер `ada_regs.h` на реальной карте, читая регистр прямо из
Windows. Из user-space MMIO недоступен — нужен инструмент с kernel-доступом.

## Данные карты (сняты с этой машины)

- GPU: RTX 4070 Super, `10DE:2783`, rev `0xA1`, subsystem `1569:F302`
- **BAR0 (регистры) = `0x52000000`, 16 MB**
- `PMC_BOOT_0` = DWORD по физ. адресу **`0x52000000`** (BAR0 + offset 0x0)

## Способ: RW-Everything

1. Скачать RW-Everything с http://rweverything.com/ , установить, запустить `Rw.exe`
   (грузит подписанный драйвер `RwDrv.sys`, систему не меняет, обратимо).
2. Меню → **Memory** (или кнопка "Memory").
3. В адрес ввести `0x52000000`, режим чтения **DWORD**.
4. Записать прочитанное 32-битное значение.

Альтернатива — вкладка **PCI**: найти Bus/Dev `01:00.0` (NVIDIA), посмотреть BAR0 в
config space (offset 0x10) → подтвердить базу `0x52000000`, затем читать память.

## Декод значения

Раскладка `NV_PMC_BOOT_0` (см. `ada_regs.h`, подтверждена драйвером Linux nova-core):

```
arch    = (value >> 24) & 0x1F     # ожидаем 0x19 = Ada
impl    = (value >> 20) & 0x0F     # ожидаем 0x4  = AD104
chipset = (value >> 20) & 0x1FF    # ожидаем 0x194 = AD104
major   = (value >> 4)  & 0x0F
minor   =  value        & 0x0F
```

Пример: если RW-Everything показал `0x19410B00`:
- arch    = (0x19410B00 >> 24) & 0x1F  = `0x19` → **Ada** ✓
- chipset = (0x19410B00 >> 20) & 0x1FF = `0x194` → **AD104** ✓

> Декодер уже сверен с авторитетным источником — драйвером Linux **nova-core**
> (chipset 0x192 = AD102; ряд Ada: AD104 = 0x194). Так что чтение через
> RW-Everything — финальная формальность: убедиться, что физический регистр
> отдаёт `arch=0x19`. Логика декода от значения не зависит, она уже верна.

## Что это доказывает

Если `arch == 0x19` — раскладка полей в `ada_regs.h` верна, и kext/dext на macOS
прочитают то же значение тем же декодером. Самый рискованный кусок месяца 1
подтверждён **без macOS**.

Если `arch != 0x19` — поправить shift/mask в `ada_regs.h`, сверив с `dev_boot.h`
из open-gpu-kernel-modules. Адрес и логика чтения остаются прежними.

## Куда записать результат

Сырое значение и разбор — в `docs/hw-dumps/` (например
`pmc-boot0-windows-rweverything.txt`) и отметить в `docs/testbed.md`.
